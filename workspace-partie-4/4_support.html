Objectif Global

L'objectif global dans ce workspace est d'apprendre à débuguer une page HTML et du code Javascript.

En ce sens, vous allez devoir travailler sur la structure HTML de la page 4_code_to_debug.html qui est une page où le code Javascript ne fonctionne pas.

A la fin de ce workspace, vous aurez manipulé la console tool développeur de votre navigateur.

Tâche 1

Examiner le rendu d'une page HTML 4_code_to_debug.html à l'aide de votre navigateur

Question

Qu'est-ce qu'il y a sur cette page?

Il y a un bouton dont le clic est censé afficher la valeur d'une variable x inconnue et un lien pour retourner au workspace.

Dans l'état actuel, si vous cliquez sur le bouton Clique ici pour afficher la valeur de la variable x , rien ne se passe.

Tâche 2

Apprendre à debuguer du code

Modifiez le code de la page 4_code_to_debug.html pour y rajouter un message dans la console devtool (CTRL+SHIFT+K sur Firefox) à l'aide de la méthode console.log("...") . Rajoutez ce message avant le code document.body.innerHTML = "<p>La valeur de x = " + x + "</p>";

Question 1

Pourquoi place-t-on le message à afficher sur la console devtool avant ce code?

On place le message à afficher sur la console devtool avant ce code pour tester si le code de addXToThePage est bien executé lors du clic du bouton.

Tâche 3

Examiner les messages de console devtool

Après avoir rajouter la méthode console.log("...") au bon endroit dans le code de la page, rafraîchissez la page tout en gardant la console devtool ouverte et appuyez de nouveau le bouton Clique ici pour afficher la valeur de la variable x

Question

Qu'obtenez-vous sur la console devtool?

Le log ne se fait pas et une erreur ReferenceError est signalée. En effet, la variable javascript addXToToThePage n'existe pas, il y a une typo. Seule la référence addXToThePage existe. C'est la fonction qui a été définie dans notre script.

Tâche 4

Examiner et corriger du code bugué

Examinez attentivement le code de la page 4_code_to_debug.html au niveau de la définition, la déclaration et l'appel de la fonction javascript de la page

Question 1

Comment la fonction a été déclaré?

Elle a été déclarée dans une balise scipt avec comme nom addXToThePage.

Question 2

Comment la fonction a été défini?

Elle est définie par une séquence de deux instructions une qui mute le contenu HTML du body et l'autre qui met bizarrement à jour la variable avec encore une typo sur un x. En revanche, la variable x n'étant pas déclarée dans le corps de la fonction, ce serait bien la variable x déclarée en dehors de la fonction (donc globalement) de valeur initialisée à 10 qui serait mise à jour.

Question 3

Comment la fonction a été appelé?

Avec la typo précitée: addXToToThePage un To de trop.

Tâche 5

Corriger du code bugué

Corrigez l'erreur au niveau de cette fonction.

Tâche 6

Deuxième correction

Malgré la correction que vous venez d'apporter, il reste une deuxième erreur.

Examinez la valeur de la variable x lors de son initialisation à l'aide d'un éditeur de texte.

Question 1

Quelle est la valeur de x à son initialisation?

x vaut 10 lors de son initialisation.

Question 2

Comment la fonction devrait normalement modifier la variable x d'après la documentation?

x = x +1

Question 3

Pourtant, quelle est la valeur de x qui s'affiche lorque vous cliquez sur le bouton ?

La valeur est 10, ce qui est normal puisqu'elle est affichée avant d'être modifiée.

Tâche 7

Corriger du code bugué

Modifiez le code de la page pour qu'il n 'y ait plus d'erreur. Puis, rafraîchissez la page et amusez-vous plusieurs fois en appuyant sur le bouton pour afficher les valeurs de x

Question

Quel est le comportement de la page après avoir corrigé les deux erreurs?

Si on ne corrige que les deux typos le bouton disparaît. Pour avoir le comportement qui semble désiré il faut concaténer le HTML de la balise body avec celui qu'il contient déjà.
