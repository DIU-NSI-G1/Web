Objectif Global

L'objectif global de ce workspace est d'apprendre à modifier la structure d'une page HTML existante.

En ce sens, la structure HTML de ce support nommé 1_support.html ressemble à une page d'un site Web classique.

Vous allez modifier la structure HTML de ce support au fil des tâches à réaliser.

A la fin de ce workspace, il est impératif que vous ayez compris dans sa globalité la structure HTML et le style CSS de chaque section de ce support.

Ainsi vous aurez plus de facilité à modifier les supports HTML des autres workspaces pour répondre aux questions et vous aurez également plus de facilité à modifier les autres fichiers HTML et CSS.

Question 1

A quoi correspond chacun de ces fichiers HTML?

Réponse

Réponse

  • Le fichier card.html

    Ce fichier correspond à une partie de la page contenant les données organisées en articles et composées de titre(s) et de structures de données type paragraphe(s), tableau(x)... Ce type de fichier permet (entre-autre) de structurer des articles de blogs, des messages de forums, des commentaires...

    • Le fichier nav.html

    Ce fichier correspond à une zone structurée de navigation dans la page. Cette partie fournie la structure de base permettant de créer un menu "hyperlink", un ensemble d'objet permettant la navigation ... Ce fichier ne permet d'afficher que des liens.

  • Le fichier header.html

    Ce fichier correspond à une zone d'entête de page HTML. Il comprend un/des titre(s) et une/des structure(s) de données type pargraphes, tableau(x) ...

  • Le fichier footer.html

    Ce fichier correspond à une zone de pied(s) de page HTML. Il comprend un/des titre(s) et une/des structure(s) de données type pargraphes, tableau(x) ...
Chacun de ces fichier ne contient qu'un élément d'une partie de page html. Le contenu de ces fichiers est généralement formé des contenus de tous ces fichiers.

Question 2

Expliquez le rendu à l'aide du navigateur de chacun de ces fichiers

Réponse

Dans chacun de ces fichiers, on remarque qu'aucun style n'a été appliqué. C'est ce qu'on appelle aussi un rendu "HTML brut". En effet, pour les fichiers card.html, header.html et footer.html, chacun d'eux contient un titre écrit en plus gros que le reste grâce au container <hn> </hn> une partie de document non formatée et un lien. Seul le lien ressort de manière différente. On remaque que dans ces fichiers si on retire les containers <article> ... </article> , <header> ... </header> et <footer> ... </footer> il n'y a aucun effet visuel. Docn ces containers sont là pour structurer le code et par la suite permettre l'application de styles grâce à CSS.

Dans le fichier nav.html (sous Firefox) on constate que le retrait des balises <nav> ... </nav> a pour effet de supprimer la zone réservée aux liens permettant la navigation.

En conclusion, ces quatre fichiers conrrespondent à quatre parties différentes d'une même page Web qu'on retrouve presqu'à chaque fois: Pieds de page, Entêtes de pages, Zones de menus et Zone de données contenant les données de la page. Ces fichiers contiennent un balisage qui n'a pas pour intérêt de modifier l'apparence de la page mais permet de la structurer. En effet, c'est à partir de cette structure qu'on pourra déterminer un style (présentation) ou alors appliquer un un programme type javascript qui permettra d'agir dynamiquement sur la page.

Question 1

Quelles sont ces balises communes?

Réponse

Les balises communes à chaque fichier sont:

<!DOCTYPE html> qui indique qu'il s'agit d'un document type HTML 5

<html> et </html> définissent respectivement le début et la fin de la page HTML.

<head> et </head> indiquent la zone d'entête de la page HTML où on peut passer des informations indiquant le titre de la page, le type d'encodage, les liens vers des fichiers CSS, Javascript ...

<title> et </title> permet de donner le titre de la page qui s'affichera dans l'onglet correspondant. <meta charset="UTF-8"> est située dans la zone d'entête et permet de définir le type d'encodage (ici UTF-8).

On peut toutefois remarquer une différence entre le code <meta http-equiv="content-type" content="text/html; charset=UTF-8"> de 1_support.html et <meta charset="UTF-8"> des fichiers de la zone brique sans CSS, mais en réalité il s'agit de la même chose. En effet, <meta http-equiv="content-type" content="text/html; charset=UTF-8"> indique le charset dans la version de XHTML ou HTML4.01, tandis que <meta charset="UTF-8"> correspond à la même chose en HTML5.

<body> et </body> définissent le corps de la page HTML qui contient l'ensemble des données structurées. Par données on entend: texte, son, images, vidéos, liens ...

<a href = "..."> {Texte du lien} </a> indique un lien permettant la navigation. Une page HTML sans line est inutile car elle ne permet pas la navigation, sauf dans le cas très particulier où le site est composé d'une page unique. Même s'il es possible de réaliser des liens à l'aide du DOM de Javascript, l'usage des balise HTML pour les liens est toutefois la méthodes la plus répandue. De plus, les "bots" des moteurs de recherche ne prennent pas toujours en compte de la même manière les liens créés à l'aide de Javascript, ce qui peut rendre le référencement du site plus difficile.


<nav>nav; et </nav> permet de structurer une zone de navigation web formée de liens hypertexte.

<header> et </head> définit une zone d'entête de page.

<footer> et </footer> définit une zone de pied de page.

<article> et </article> définit une partie de zone de données correspondant à un article (Elément de forum, article de Blog...).

<hn> et </hn> est la seule partie de la section brique sans CSS qui permette un formatage visuel de la page. Mais ce formatage est avant tout structurant.
<a> et </a> définit un lien hypertexte, soit vers une zone du document courant identifiée par un id, soit vers un autre document. <p> et </p> définit un paragraphe incluant un saut de ligne avant et après la partie délimitée par ces deux balises/

Seules certaines balises parmi toutes les balises utilisées sur cette page servent à structurer ce support en parties distinctes. Lesquelles de ces balises communes à celle du répertoire brique_sans_css vous semblent former le squelette de la page?

Réponse

On peut distinguer deux types de squelettes. Celui qui est commun à toutes les pages et celui qui permet une structuration spécifique à la page courante.
  • Squelette commun à toutes les pages Web:

    Ce squelette est formé des par les balises <html> , <head> , <meta> , <title> et <body> (accompgnées par leurs balises fermantes et constitue l'ossature de toutes les pages HTML.


  • Squelette commun à la page courante et à la partie brique sans CSS:

    Ce squelette est formé par les balises <hn> , <a> , <p> , <nav> , <article> , <header> et <footer> (et des balises fermantes correspondantes) qui ont pour effet de structurer la page 1_support.html.

Question 1

Le pied de page HTML que vous venez de rajouter n'avez pas de style CSS. Toutefois, après l'avoir inclu dans le support, un style lui a été appliqué. Comment ce style lui a été appliqué?

Réponse

Dans le fichier footer.html, on n'a pas lié, dans la partie <head> ... </head> , la page courante au fichier CSS à l'aide du tag <link rel="stylesheet" href="../workspaces-css/style_supports.css">. Comme il n'y a pas non plus de balises <style> ou d'attribut "style = ..." dans un tag, aucun style ne peut être appliqué.

Par contre dans le fichier 1_support.html il y a ce "link" donc tous les styles de ce fichier seront appliqués à la page. Ainsi, comme le tag footer contient l'attribut class = "footer" et que l e fichier style_supports.css contient la définition de la classe .footer{ ... }, le style décrit dans cette classe est appliqué à ce deuxième pied de page.

Question 2

Voyez-vous un lien entre la brique que vous avez rajouté et le pied de page utilisé dans ce support? Justifiez.

Réponse

Le lien entre la brique rajoutée et la pied de page se trouve dans le code suivant:

<footer class="footer">
<h2>Contact</h2>
<p class="answer">Nom - prénom - prenom.nom@email.com</p>
</footer>

Le tag footer structure le pied de page. l'attribut class = "footer" applique le style CSS défini dans la classe footer du fichier style_supports.css. Le tag h2 permet d'écrire le titre du pied de page et le tag p défini le paragraphe du pied de page. Or le pied de page du support est bati sur le modèle donné dans footer.html. C'est l'ensemble de ces élément qui permet de créer un pied de page et c'est ce qui est utilisé dans le support.

Question

Pourquoi rien ne passe lorsque vous cliquez sur les liens de navigation de votre nouveau menu de navigation?

Réponse

Les liens vers des pages ne fonctionnent pas car ces pages n'existent pas et lorsqu'on clique dessus le lien pointe vers une page insexistante.
La deuxième catégorie de liens pointe vers un identifiant de la page. Comme cet identifiant n'existe pas dans le support, le lien n'a aucun effet. Il faudrait donc écrire href=# suivi de l'identifiant d'un tag dans la page.

Question

Comment fonctionne le menu de navigation initial de la page?

Réponse

Le menu de navigation initial de la page fonctionne de la manière suivante:

Les options de menu intitulées HEADER, NAV, CARD et ACCUEIL sont un lien vers un autre fichier dont le chemin local est indiqué dans l'attribut href de la balise <a>.
Les autres options de menu pointent sur un identifiant de la page indiqué par un "#". Cet identifiant est déclaré comme attribut de tag et est unique. Par exemple la tâche 1 commence par le tag:
<h2 id="T1">Tâche 1</h2>. L'id est ici T1. Dans la zone menu, le lien contient l'attribut id="#T1" le # indique qu'il s'agit d'un attribut dans la page et non d'un chemin.
Ainsi, lorsqu'on clique sur le lien dont le href est un identifiant, "l'ancre" de la page de déplace jusqu'à l'identifiant. Lorsqu'on clique sur un lien, on change de page.

Question

Il y a-t-il une différence entre une balise HTML div de classe card et une balise HTML article de classe card utilisé dans ce support? Justifiez.

Réponse

Visuellement, on ne remarque aucune différence entre une balise article de classe card et une balise div de classe card.
D'un point de vue plus structurel, on peut noter une différence sensible entre ces deux balises. D'une part, la balise article segmente des parties du document et a un apport sémantique fort.
D'autre part, la balise div a un role plus neutre et se contente de regrouper des éléments HTML la plupart du temps dans le but de leur appliquer des CSS.
Comme la même classe card est appliquée à ces deux balises, visuellement on aura le même rendu.

Question 1

Que fait le style CSS concernant l'en tête ?

Réponse

Le CSS qui définit le style de l'entête permet de centrer le texte (text-align: center;), choisit la couleur du font d'écran de l'entête (background: white;) et définit un padding (cf définition de padding dans la section footer - padding: 30px;) de 30 pixels. De plus le code : header h1 { font-size: 50px; } permet de choisir la taille - en pixels - des titres h1 qui suivent immédiatement un tag header.

Question 2

Que fait le style CSS concernant le menu de navigation ?

Réponse

/* Style the top navigation bar */
nav {                                 => Style appliqué à tous les containers nav
overflow: hidden;              => Ce style permet de cacher la partie du texte qui déborde de la zone de menu
background-color: #333;     => le fond d'écran prend la couleur #333
}

/* Style the topnav links */
nav a {                               => définit le style des liens dans le container nav
float: left;                           => Détermine comment un élément flottant se positionne dans la page. Ici, sur la gauche.
display: block;                   => Déternmine un style d'affichage en block
color: #f2f2f2;                    => Définit la couleur du texte dans la zone.
text-align: center;              => Centre le texte dans la zone de lien en tenant compte des paramètres de padding, border et margin.
padding: 14px 16px;         => Définit un padding de 14 pixels haut et bas et 16 pixels gauche et droite. Cf padding dans la zone footer.
text-decoration: none;       => Indique que le texte du lien ne comprend aucune décoration. Le but est de supprimer le souligné qui apparaît par défaut dans le texte du lien.
}

/* Change color on hover */
nav a:hover {                     => Permet de changer le style de tous les liens qui se trouvent dans le nav lorsque la souris survole la zone du lien.
background-color: #ddd;   => change la couleur du fond de la zone de lien lors du survole de la souris.
color: black;                       => Met le texte du lien en noir lorsque la souris survole le lien.
}

Question 3

Que fait le style CSS concernant le pied de page ?

Réponse

Le CSS créer une classe footer. Celle-ci a pour effet de centrer le texte dans le footer (text-align: center;). De plus, le style CSS permet de choisir la couleur du font de valeur #ddd (background: #ddd;) du footer, une marge horizontale supérieure de 20 pixels (margin-top: 20px;) et un écart de remplissage (padding) sur 4 côtes de 20 pixels (padding: 20px;).
